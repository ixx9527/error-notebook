# 错题本小程序 — 产品方案与交互设计

## 一、产品概述

### 1.1 产品定位

**智能错题整理与科学复习工具**，面向小学生家长群体。

核心价值：拍照即录入 → AI 自动识别 → 艾宾浩斯曲线复习 → 数据驱动查漏补缺。

### 1.2 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 主要用户 | 小学生家长（25-45 岁） | 高效整理错题、督促复习、看到学习进展 |
| 次要用户 | 学生本人（高年级） | 自主查看错题、复习打卡 |

### 1.3 使用场景

1. **晚间作业后**：家长拍错题照片，AI 自动录入，2 分钟完成
2. **碎片时间**：通勤/午休时查看今日待复习题
3. **周末复习**：集中完成本周待复习题目，查看统计报告
4. **考前冲刺**：导出 PDF 错题集，打印练习

### 3.2 上传页（upload）

**目标**：最快路径完成「拍照 → 看到识别结果 → 确认保存」。

#### 交互流程

```
步骤 1：选择图片          步骤 2：AI 识别中          步骤 3：确认结果
┌──────────────┐     ┌──────────────┐     ┌──────────────────────┐
│              │     │              │     │  科目：[数学 ▼]        │
│  ┌────────┐  │     │   ◐ 识别中... │     │  知识点：分数加减法     │
│  │        │  │     │              │     │                      │
│  │  预览   │  │ ──→ │  AI 正在分析  │ ──→ │  题目：               │
│  │  图片   │  │     │  题目内容...  │     │  ┌────────────────┐  │
│  │        │  │     │              │     │  │ 1/3 + 2/5 = ?  │  │
│  └────────┘  │     │  预计 10 秒   │     │  └────────────────┘  │
│              │     │              │     │                      │
│ [拍照] [相册]│     │  ░░░░░░ 60%  │     │  学生答案：3/8         │
│              │     │              │     │  正确答案：11/15       │
└──────────────┘     └──────────────┘     │  错因：通分错误        │
                                          │  分析：未找最小公倍数   │
                                          │                      │
                                          │  [编辑] [确认保存 ✓]   │
                                          └──────────────────────┘
```

#### 详细交互

| 阶段 | 交互 | 说明 |
|------|------|------|
| 选图 | 点击「拍照」/「从相册选择」 | `wx.chooseMedia`，限制 1 张 |
| 预览 | 可缩放查看原图 | 确认图片清晰 |
| 上传 | 自动调用异步上传 | `POST /errors/upload/async` |
| 等待 | 显示进度动画，每 2 秒轮询 | `GET /errors/upload/status/{task_id}` |
| 结果 | 展示 AI 识别的所有字段 | 结构化展示 |
| 编辑 | 点击「编辑」进入编辑模式 | 可修改所有识别字段 |
| 保存 | 点击「确认保存」 | `PUT /errors/{id}` 提交修改（如有） |
| 授权 | 保存后请求订阅消息授权 | 用于复习提醒 |

#### 编辑模式字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 科目 | 选择器 | 是 | 语文/数学/英语/其他 |
| 知识点 | 文本输入 | 否 | AI 识别的 topic |
| 题目内容 | 多行文本 | 是 | AI 识别的 question_text |
| 学生答案 | 文本输入 | 否 | — |
| 正确答案 | 文本输入 | 否 | — |
| 错因分类 | 文本输入 | 否 | 如"计算错误""概念混淆" |
| 错因分析 | 多行文本 | 否 | AI 生成的分析 |
| 标签 | 标签输入 | 否 | 自定义标签 |

#### 编辑模式

点击「编辑」后，所有文本字段变为可编辑状态：
- 科目 → picker 选择
- 知识点/题目/答案/错因 → textarea
- 标签 → 标签编辑
- 底部出现「保存」和「取消」按钮
- 保存调用 `PUT /errors/{id}`

#### 更多操作（⋯ 菜单）

| 选项 | 行为 |
|------|------|
| 删除 | 二次确认 → `DELETE /errors/{id}` |
| 归档 | 标记为 archived |
| 分享 | 生成图片分享 |

## 四、核心用户流程

### 4.1 新用户首次使用

```
打开小程序
  │
  ▼
微信授权登录（静默） ──→ 失败 → 账号密码登录/注册
  │
  ▼（成功）
进入首页（空状态引导）
  │
  ▼
「添加第一个孩子」引导弹窗
  │
  ▼
「拍一道错题试试」引导 → 上传页
  │
  ▼
完成首次上传 → 看到 AI 识别结果 → 确认保存
  │
  ▼
返回首页 → 看到数据更新 → 引导复习
```

### 4.2 日常使用流程

```
每日循环：
  打开小程序 → 首页看今日待复习数
    │
    ├─ 有作业 → 拍照录入新错题
    │
    └─ 有待复习 → 进入复习 → 逐题复习 → 打卡完成
                                        │
                                        ▼
                                   返回首页看统计更新
```

### 4.3 周期性流程

```
每周：查看统计 → 发现薄弱科目 → 针对性录入
每月：导出月度报告 → 打印/分享
考前：导出错题 PDF → 打印练习
```
