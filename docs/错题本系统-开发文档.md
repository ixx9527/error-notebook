# 小学错题本系统 - 开发文档

> 基于 Qwen-VL 的智能错题录入、复习提醒与导出系统

---

## 一、项目概述

### 1.1 背景

小学生错题管理痛点：

| 痛点 | 现有方案问题 |
|------|-------------|
| 错题录入 | 作业帮扫描后格式乱、图形变形 |
| 类似题推荐 | 质量差，不实用 |
| 复习提醒 | 无间隔复习机制，不符合艾宾浩斯记忆法 |
| 最终载体 | 希望用实体本子，但需要数字化录入和排版 |

### 1.2 解决方案

**核心思路**：用 Qwen-VL 一个模型替代 OCR + 版面分析 + 公式识别 + 手写识别 + 图形重绘 + 错因分析的全链路。

```
微信小程序（前端）→ 后端服务（FastAPI）→ Qwen-VL（AI引擎）
                                         ↓
                         存储 → 艾宾浩斯复习计划 → PDF导出 + 消息提醒
```

### 1.3 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | 微信小程序 | 拍照、浏览、导出 |
| 后端 | Python + FastAPI | API 服务、业务逻辑 |
| AI 引擎 | Qwen-VL（DashScope API） | 图片理解 + 结构化输出 |
| 图像处理 | OpenCV（Python） | 梯形矫正（可选增强） |
| 数据库 | PostgreSQL | 错题、复习计划、用户数据 |
| 文件存储 | 本地磁盘 / OSS | 原图、处理后的图片 |
| PDF 导出 | WeasyPrint / Puppeteer | 结构化排版输出 |
| 定时任务 | APScheduler | 艾宾浩斯复习检查 |
| 消息推送 | Server酱 / 企业微信机器人 | 微信通知 |

## 三、Qwen-VL 核心引擎

### 3.1 调用方式

通过阿里云 DashScope API 调用 Qwen-VL：

```python
import dashscope
from dashscope import MultiModalConversation

dashscope.api_key = "your-api-key"

def analyze_error_image(image_path: str) -> dict:
    """分析错题图片，返回结构化结果"""
    
    prompt = """你是一个小学错题整理助手。请仔细分析这张错题照片，输出以下JSON格式的结果（只输出JSON，不要其他内容）：

{
  "subject": "语文|数学|英语|其他",
  "topic": "具体知识点，如：三位数乘法、阅读理解-主旨概括",
  "question_text": "完整的题目正文，保留题目原有的结构和编号",
  "formulas": ["如果题目包含数学公式，用LaTeX表示，如 $235 \\times 12 = ?$"],
  "figures": [
    {
      "description": "图形的文字描述",
      "svg": "用SVG代码重新绘制该图形，保持原始比例和标注"
    }
  ],
  "student_answer": "学生的原始答案（手写内容识别）",
  "correct_answer": "正确答案",
  "error_type": "错因分类，如：计算错误、概念混淆、审题不清、格式错误",
  "error_analysis": "详细的错因分析，说明为什么错",
  "tags": ["标签1", "标签2", "标签3"]
}

要求：
1. 图形部分必须用标准SVG代码重绘，保持原始比例、标注和虚线/实线
2. 如果照片中有多个题目，拆分成多个对象，放在数组中返回
3. 无法识别或不确定的字段填 null
4. 数学公式用LaTeX格式
5. 只输出JSON，不要任何解释或markdown格式标记"""

    messages = [
        {
            "role": "user",
            "content": [
                {"image": f"file://{image_path}"},
                {"text": prompt},
            ]
        }
    ]

    response = MultiModalConversation.call(
        model="qwen-vl-max",  # 或 qwen-vl-plus，根据需要选择
        messages=messages,
    )

    return parse_response(response)


def parse_response(response) -> dict:
    """解析 Qwen-VL 返回的内容，提取 JSON"""
    content = response.output.choices[0].message.content[0]["text"]
    
    # 清理可能的 markdown 代码块标记
    content = content.strip()
    if content.startswith("```json"):
        content = content[7:]
    if content.startswith("```"):
        content = content[3:]
    if content.endswith("```"):
        content = content[:-3]
    
    return json.loads(content.strip())
```

### 3.4 调用注意事项

| 事项 | 说明 |
|------|------|
| 模型选择 | `qwen-vl-max`：最强效果；`qwen-vl-plus`：性价比更高 |

### 3.5 JSON 解析容错

```python
import json
import re

def safe_parse_json(text: str) -> dict | list | None:
    """多策略 JSON 解析容错"""
    # 策略1：直接解析
    try:
        return json.loads(text)
    except:
        pass
    
    # 策略2：去掉 markdown 代码块标记
    cleaned = re.sub(r'^```(?:json)?\s*', '', text.strip())
    cleaned = re.sub(r'\s*```$', '', cleaned)
    try:
        return json.loads(cleaned)
    except:
        pass
    
    # 策略3：提取第一个 { 或 [ 到最后一个 } 或 ]
    match = re.search(r'[\[{].*[\]}]', text, re.DOTALL)
    if match:
        try:
            return json.loads(match.group())
        except:
            pass
    
    # 策略4：重试（重新调用 Qwen-VL）
    return None
```

## 四、图像处理（可选增强）

### 4.1 梯形矫正

Qwen-VL 对轻微形变有较好的容错能力。但如果拍照角度倾斜严重，建议先做梯形矫正：

```python
import cv2
import numpy as np

def correct_perspective(image_path: str, output_path: str) -> str:
    """对照片做梯形矫正"""
    img = cv2.imread(image_path)
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    gray = cv2.GaussianBlur(gray, (5, 5), 0)
    
    # 边缘检测
    edged = cv2.Canny(gray, 75, 200)
    
    # 查找轮廓
    contours, _ = cv2.findContours(edged, cv2.RETR_LIST, cv2.CHAIN_APPROX_SIMPLE)
    contours = sorted(contours, key=cv2.contourArea, reverse=True)[:5]
    
    # 找到四边形轮廓
    screen_cnt = None
    for c in contours:
        peri = cv2.arcLength(c, True)
        approx = cv2.approxPolyDP(c, 0.02 * peri, True)
        if len(approx) == 4:
            screen_cnt = approx
            break
    
    if screen_cnt is None:
        # 未检测到四边形，返回原图
        return image_path
    
    # 透视变换
    pts = screen_cnt.reshape(4, 2)
    rect = order_points(pts)
    (tl, tr, br, bl) = rect
    
    width = max(int(np.sqrt((br[0]-bl[0])**2 + (br[1]-bl[1])**2)),
                int(np.sqrt((tr[0]-tl[0])**2 + (tr[1]-tl[1])**2)))
    height = max(int(np.sqrt((tr[0]-br[0])**2 + (tr[1]-br[1])**2)),
                 int(np.sqrt((tl[0]-bl[0])**2 + (tl[1]-bl[1])**2)))
    
    dst = np.array([
        [0, 0],
        [width - 1, 0],
        [width - 1, height - 1],
        [0, height - 1]
    ], dtype="float32")
    
    matrix = cv2.getPerspectiveTransform(rect, dst)
    warped = cv2.warpPerspective(img, matrix, (width, height))
    
    cv2.imwrite(output_path, warped)
    return output_path


def order_points(pts):
    """将四个点按 左上、右上、右下、左下 排序"""
    rect = np.zeros((4, 2), dtype="float32")
    s = pts.sum(axis=1)
    rect[0] = pts[np.argmin(s)]  # 左上：和最小
    rect[2] = pts[np.argmax(s)]  # 右下：和最大
    diff = np.diff(pts, axis=1)
    rect[1] = pts[np.argmin(diff)]  # 右上：差最小
    rect[3] = pts[np.argmax(diff)]  # 左下：差最大
    return rect
```

### 4.2 图片预处理流程

```python
def preprocess_image(image_path: str) -> str:
    """上传前的图片预处理"""
    img = cv2.imread(image_path)
    
    # 1. 尺寸压缩（Qwen-VL 不需要超高清图）
    h, w = img.shape[:2]
    max_dim = 1920
    if max(h, w) > max_dim:
        scale = max_dim / max(h, w)
        img = cv2.resize(img, (int(w * scale), int(h * scale)))
    
    # 2. 梯形矫正（可选）
    processed_path = image_path.replace('.', '_processed.')
    cv2.imwrite(processed_path, img)
    corrected_path = correct_perspective(processed_path, 
                                         processed_path.replace('.', '_corrected.'))
    
    return corrected_path
```

### 4.3 使用建议

- **默认流程**：直接把原图发给 Qwen-VL，大部分场景已经够用
- **增强流程**：先做梯形矫正，再发给 Qwen-VL，适合拍照角度较大的情况
- 可以提供一个「矫正开关」让用户选择

### 5.2 艾宾浩斯复习间隔

```python
EBBINGHAUS_INTERVALS = [1, 2, 4, 7, 15, 30]  # 天

def generate_review_plans(question_id: int, user_id: int, created_date=None):
    """为新增错题生成复习计划"""
    from datetime import date, timedelta
    
    created_date = created_date or date.today()
    plans = []
    
    for i, interval in enumerate(EBBINGHAUS_INTERVALS):
        review_date = created_date + timedelta(days=interval)
        plans.append({
            "question_id": question_id,
            "user_id": user_id,
            "review_date": review_date,
            "review_round": i + 1,          # 第 1~6 轮
            "interval_days": interval,
            "status": "pending",
        })
    
    return plans
```

async def complete_review(plan_id: int, body: CompleteReview):
    plan = db.get_review_plan(plan_id)
    if not plan:
        raise HTTPException(404, "复习计划不存在")
    
    # 标记完成
    db.update_review_plan(plan_id, status="completed",
                         completed_at=datetime.now(),
                         mastery_level=body.mastery_level)
    
    # 如果掌握程度低，可以在下一轮提前复习
    next_plan = db.get_next_review_plan(plan.question_id, plan.review_round + 1)
    
    return {"code": 0, "message": "复习完成",
            "data": {"next_review": next_plan}}

### 7.2 动态调整（可选增强）

```python
def get_adjusted_interval(base_interval: int, mastery_level: int) -> int:
    """
    根据掌握程度动态调整下次复习间隔
    mastery_level: 1-5（1=完全不会，5=完全掌握）
    """
    if mastery_level <= 1:
        # 掌握差：缩短间隔，提前复习
        return max(1, base_interval // 2)
    elif mastery_level <= 3:
        # 掌握一般：按标准间隔
        return base_interval
    else:
        # 掌握好：延长间隔
        return int(base_interval * 1.5)
```

async def daily_review_reminder():
    """每天早上 8:00 检查今日待复习，发送提醒"""

def format_reminder_message(reminder: dict) -> str:
    """格式化提醒消息"""
    items = reminder["items"]
    subject_counts = {}
    for item in items:
        subj = item["subject"]
        subject_counts[subj] = subject_counts.get(subj, 0) + 1
    
    subject_str = "、".join(f"{k}{v}道" for k, v in subject_counts.items())
    
    return f"""📚 今日复习提醒（{reminder['date']}）
共有 {len(items)} 道错题待复习：{subject_str}

点击查看详情并开始复习"""

## 十二、关键注意事项

### 12.1 Qwen-VL 调用优化

1. **图片压缩**：上传前压缩到 1MB 以内，减少传输时间和 token 消耗
2. **异步处理**：Qwen-VL 响应 10-30 秒，API 应返回 task_id，前端轮询结果
3. **重试机制**：JSON 解析失败时重试，最多 2 次
4. **缓存**：相同图片不重复调用

```python
# 异步处理模式
@app.post("/api/errors/upload")
async def upload_error(file: UploadFile = File(...)):
    # 立即返回 task_id
    task_id = create_task(file)
    return {"code": 0, "data": {"task_id": task_id, "status": "processing"}}

@app.get("/api/errors/upload/status/{task_id}")
async def check_status(task_id: str):
    task = get_task(task_id)
    return {"code": 0, "data": task}
```

- 图片访问需要鉴权，不能直接暴露文件路径
- 定期清理已删除错题的图片文件
